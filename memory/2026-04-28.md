## Pinterest→GIF自动化脚本 v10（pinterest_dl.py）

- **脚本位置**：`E:\workbuddy\huaban\pinterest_dl.py`
- **Python**：venv `C:\Users\Administrator\.workbuddy\binaries\python\envs\default\`（含playwright）
- **bat**：`E:\workbuddy\huaban\run_pinterest.bat`（已更新）
- **输出**：`E:\workbuddy\huaban\HuaBan\pinterest动图XX/`（自动递增）

### 核心流程（v10最终方案）
1. 从搜索页提取pin ID列表（`/pin/数字` 链接）
2. 逐个打开pin详情页，从DOM提取HD图片URL（`/736x/`、`/originals/`）
3. 浏览器直接打开HD URL（携带cookies）→ canvas提取图片数据
4. 类型检测 → 分类保存 → GIF转换

### 关键技术踩坑
- **403问题**：Pinterest CDN URL带签名，直接Python请求被拒。解决：从pin详情页内提取URL，浏览器环境携带cookies，canvas提取绕过CDN签名限制
- **v7 vs v10**：v7用Python urllib下载（有部分成功），v10用浏览器canvas提取（成功率更高，图片更大）
- **缩略图升级失败**：从`/236x/`到`/736x/`的URL替换不能直接用（签名不兼容），必须从详情页重新获取原始HD URL
- **Playwright evaluate语法**：`page.evaluate()`内不能有`return`语句在函数外；用IIFE `(function(){...})()`解决
- **Python raw string**：JS正则里`\/`在raw string里没问题，但字符串内的`'`和`"`需要正确配对
- **类型检测**：GIF判断用`n_frames > 1`；Pinterest视频缩略图（`/videos/thumbnails/originals/`）是JPG，不是真正的动图

### 批次文件夹命名规则
- 格式：`pinterest动图01/`、`pinterest动图02/`...（自动找最大编号+1）
- 子目录：`dynamic_raw/`（动图原文件）/`static_raw/`（静态原文件）/`GIF_output/`（输出GIF）
