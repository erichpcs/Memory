- 位置：`C:\Users\Administrator\.lark-cli\config.json`（WorkBuddy系统目录外）
- 认证：appId + appSecret（存在Windows系统Keychain）
- 共享性：任何以Administrator身份运行的工具都可以读取，无需重新认证

**Codex/晓超接入Feishu的可行路径：**
1. Codex如果以同一Windows用户运行 → 自动继承飞书权限
2. 验证命令：`lark-cli config show` → 看是否显示app_id
3. 若未显示 → 手动指定config路径：`lark-cli config set --config "C:\Users\Administrator\.lark-cli\config.json"`
4. 权限检查：确认Feishu Bot是否已加入目标群聊、是否有文档读写权限
5. 若无Bot权限 → 在飞书开放平台给App补充权限，无需重新认证

**不需要重走完整认证流程（device flow/扫码），因为：**
- 认证凭据已在系统Keychain中
- config.json已在全局可访问位置
- lark-cli是npm全局安装工具，非WorkBuddy专属
