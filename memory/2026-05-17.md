## 2026-05-17 feishu-cloud-docs Skill v1.1 修复（drive +import）

### 问题
用户要求在指定文件夹内创建云文档，但文档始终跑到"墨麟集团-陈昊鹏"根目录。用户极其不满。

### 根因分析
- `lark-cli docs +create` 内部调用飞书 MCP 工具（`https://mcp.feishu.cn/mcp`）
- `--folder-token` 参数在 MCP 层被**完全忽略**
- 文档始终创建在用户个人空间根目录，无法指定文件夹

### 正确方法
`lark-cli drive +import --file <md> --folder-token <token> --type docx --name <name>`

**API流程**：
1. POST `/open-apis/drive/v1/medias/upload_all` → 获取 file_token
2. POST `/open-apis/drive/v1/import_tasks` → 创建导入任务，`point.mount_key` = folder_token
3. 轮询导入状态 → 文档直接出现在目标文件夹

### 验证
- ✅ `drive files list --params '{"folder_token":"A9V9fIXgmlB1j8dqA3Jc1EGqnWg"}'` → 能列出文档
- ✅ `drive files list --params '{"folder_token":""}'` → 空（不在根目录）
- ✅ parent_token 匹配目标文件夹 token

### 面包屑路径误解
飞书文档页面显示的面包屑（"墨麟集团 > 陈昊鹏"）是**空间归属（owner）**，不是文件夹层级。所有文档面包屑都一样，包括用户自己创建的"Spine 动画需求文档"。
